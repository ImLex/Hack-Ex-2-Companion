TRAKKER 3 - COMPLETE PROJECT SUMMARY
====================================

PROJECT GOAL
------------

Trakker 3 is a local Android application inspired by Hack Ex / Hack Ex 2.

It is NOT a hacking tool.

It is a tracker/database for keeping information about discovered targets in-game.

The goal is to replace spreadsheets and notebooks with a polished intelligence database that can:

• Store targets
• Track logs
• Parse logs
• Track wallets
• Track IP addresses
• Track installed software
• Track crypto history
• Rank targets automatically
• Search everything instantly
• Analyze target value
• Become a complete offline intelligence system


==========================================================
GENERAL DESIGN PHILOSOPHY
==========================================================

The application should be designed with future expansion in mind.

Avoid temporary solutions.

Before adding features:

• Think about how future features connect.
• Prevent breaking existing systems.
• Avoid duplicated data.
• Keep database structure clean.
• Keep relationships normalized.
• Store information where it belongs.
• Make everything searchable.
• Prefer scalable systems over quick fixes.


Development style:

The user prefers:

❌ Small fixes repeatedly
❌ Rewriting the same files
❌ Temporary patches

Instead:

✅ Complete one file at a time
✅ Think ahead before coding
✅ Build systems properly once
✅ Use copy/paste ready complete files
✅ Minimize future breaking changes


==========================================================
DATABASE STRUCTURE
==========================================================


TargetEntity
------------

Represents the target identity.

Should remain mostly static.

Fields:

id
name
device
dateAdded
attackCount


Purpose:

Stores who/what the target is.

Does NOT store constantly changing information.


----------------------------------------------------------


TargetInfoEntity
----------------

Dynamic target intelligence.

Fields:

targetId
level
crypto
cryptoExtractedTotal
cryptoExtractedToday
activity
potentialScore
tags
notes
lastUpdated


Purpose:

Stores information that changes over time.

Examples:

Level changes.
Crypto changes.
Activity changes.
Score changes.


----------------------------------------------------------


IpRelationEntity
----------------

Stores discovered IP addresses.

Fields:

id
targetId
address
status
foundFromLogId
source
discoveredAt


Purpose:

Every discovered IP belongs to one target.


----------------------------------------------------------


WalletEntity
------------

Stores discovered wallets.

Fields:

id
targetId
displayAddress
fullAddress
cracked
foundFromLogId


Purpose:

Tracks wallet discovery and cracking status.


----------------------------------------------------------


LogEntity
---------

Stores imported raw logs.

Fields:

id
targetId
rawLog
timestamp
eventType
cryptoExtracted
extractedIpCount
extractedWalletCount
extractedIps
extractedWallets
parserConfidence


Purpose:

Every event should be traceable back to the original log.


----------------------------------------------------------


CryptoHistoryEntity
-------------------

Stores every crypto extraction event.

Fields:

id
targetId
amount
date
source
sourceLogId


Purpose:

Never only store totals.

Keep history for:

• Graphs
• Timeline
• Daily income
• Monthly income
• Statistics


----------------------------------------------------------


CryptoHistoryWithLog
--------------------

Relation:

CryptoHistoryEntity

+

LogEntity


Allows:

Viewing extraction history with the original event.


----------------------------------------------------------


SoftwareEntity
--------------

Master software database.

Examples:

Firewall

Proxy

Siphon

Spam

Bypasser

Rootkit

Password Encryptor

Keygen


Purpose:

Stores available software types.


----------------------------------------------------------


InstalledSoftwareEntity
-----------------------

Stores software installed on targets.

Fields:

id
targetId
softwareId
level


Example:

Target A:

Firewall Lv25

Proxy Lv20

Siphon Lv17


----------------------------------------------------------


ReviewEntity
------------

Stores targets needing manual review.
